## A. 자주 나는 오류와 해결법

| 증상 | 원인 (쉽게) | 해결 |
|---|---|---|
| `codex` 명령을 인식 못 함 / "command not found" | 설치 위치를 컴퓨터가 아직 모름 (PATH) | **컴퓨터 재시작** 후 다시 실행. 그래도 안 되면 메뉴 `6` 진단 |
| 설치가 실패함 | 인터넷 끊김 / 일시 오류 | 인터넷 확인 후 메뉴 `3`으로 다시 설치 |
| 엉뚱한 프로그램이 깔림 | 옛날 `codex` 패키지와 혼동 | 이 키트는 올바른 `@openai/codex`만 설치하니 그대로 사용 |
| 로그인 창이 안 열림 | 방화벽/네트워크 차단 | 인터넷 확인. 화면에 나오는 주소를 직접 브라우저에 붙여넣기 |
| `429` / "Too many requests" | 잠깐 너무 많이 씀 (사용량 한도) | 화면에 표시된 시간만큼 기다린 뒤 다시 |
| Node 버전이 낮다고 함 | 오래된 Node.js | <https://nodejs.org> 에서 최신 **LTS** 재설치 (22 이상 권장) |
| 로그인하라는데 막힘 | 계정 보안 설정 | ChatGPT 계정에 **2단계 인증(MFA)** 을 켠 뒤 다시 시도 |

> 더 헷갈리면 메뉴 **`6` 진단**을 눌러 나온 화면을 캡처해 두세요. 원인 찾기가 훨씬 쉬워집니다.

## B. 자주 쓰는 명령어

> 검은 화면(터미널)에서 입력합니다. 보통은 키트 메뉴만 써도 충분합니다.

### 기본
| 명령 | 뜻 |
|---|---|
| `codex` | 대화형으로 시작 (가장 쉬움) |
| `codex "할 일 설명"` | 바로 작업 시키기 — 예: `codex "이 코드 설명해줘"` |
| `codex exec "작업"` | 질문 없이 한 번에 실행 |

### 관리
| 명령 | 뜻 |
|---|---|
| `codex --version` | 설치된 버전 확인 |
| `codex logout` | 로그아웃 |
| `npm install -g @openai/codex@latest` | 최신으로 업데이트 (메뉴 `4`와 동일) |
| `npm uninstall -g @openai/codex` | 제거 (메뉴 `7`과 동일) |

### 모드 (내 컴퓨터 보호)
| 명령 | 뜻 |
|---|---|
| `codex --sandbox read-only` | 읽기만 함 (파일 수정 안 함) |
| `codex --sandbox workspace-write` | 지금 폴더 안에서만 수정 허용 (이 키트 기본) |

### 대화 중 (Codex 켜진 상태에서)
- `/help` 입력 → 사용할 수 있는 기능 목록 보기
- `/quit` 입력 또는 `Ctrl` + `C` → 종료

## C. 메뉴 번호 빠른 참조

| 번호 | 기능 |
|---|---|
| Enter / 1 | 코덱스 시작 |
| 2 | 다른 폴더에서 시작 |
| 3 | 설치 / 다시 설치 |
| 4 | 업데이트 |
| 5 | 모드 바꾸기 (파일수정 ↔ 읽기전용) |
| 6 | 안 될 때 (자동 진단) |
| 7 | 제거 |
| H / Q | 사용법 / 끄기 |

## D. 알아두면 좋은 것

- **설정·로그인 정보가 저장되는 곳**: `C:\Users\(내이름)\.codex\` (제거 시 이 폴더를 지울지 따로 물어봅니다)
- **키트 설정 저장**: `C:\Users\(내이름)\AppData\Local\codex-cli-one-click-kit\`
- **비용**: ChatGPT 무료 계정은 제한적으로, Plus 이상은 더 많이 사용 가능. API 키를 쓰면 사용한 만큼 별도 과금됩니다.
- AI가 준 코드·명령은 **항상 직접 확인**하고, 잘 모르면 먼저 `읽기 전용` 모드로 써보세요.
